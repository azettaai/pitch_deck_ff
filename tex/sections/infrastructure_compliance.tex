\subsection{Enterprise Tier: Single-Tenant Isolation}

The Enterprise tier provides complete isolation for organizations with strict security requirements:

\subsubsection{VPC Peering Specifications}

\begin{itemize}[noitemsep]
    \item \textbf{Private Connectivity}: AWS PrivateLink / GCP Private Service Connect
    \item \textbf{Network Isolation}: Dedicated VPC with customer-controlled CIDR blocks
    \item \textbf{No Internet Exposure}: All traffic remains within cloud provider backbone
    \item \textbf{IP Whitelisting}: Customer-defined ingress/egress rules
\end{itemize}

\subsubsection{Single-Tenant Architecture}

\begin{itemize}[noitemsep]
    \item \textbf{Dedicated Compute}: Isolated VM instances (not shared with other customers)
    \item \textbf{Dedicated Storage}: Separate PostgreSQL cluster with customer-specific encryption keys
    \item \textbf{Dedicated GPU Pool}: Reserved inference capacity for predictable latency
    \item \textbf{Custom Scaling}: Customer-defined auto-scaling policies
\end{itemize}

\subsection{Security \& Data Protection}

\subsubsection{Encryption Standards}

\begin{itemize}[noitemsep]
    \item \textbf{In Transit}: TLS 1.3 for all API communications
    \item \textbf{At Rest}: AES-256 encryption with customer-managed keys (BYOK) available
    \item \textbf{Key Management}: AWS KMS / GCP Cloud KMS integration
\end{itemize}

\subsubsection{Data Isolation}

\begin{itemize}[noitemsep]
    \item \textbf{Namespace Separation}: Logical isolation at application layer (Free/Premium tiers)
    \item \textbf{Physical Isolation}: Dedicated infrastructure (Enterprise tier)
    \item \textbf{Privacy Guarantee}: Customer data is \textbf{never} used for model training
    \item \textbf{Audit Logging}: Anonymized access logs with configurable retention
\end{itemize}

\subsection{Omnilingual Data Residency}

The ``Omnilingual'' nature of Cosma introduces unique data residency considerations. A single unified index may contain text from multiple jurisdictions.

\subsubsection{GDPR-Compliant Sharding}

% TODO: Add specifics on how omnilingual indexes handle EU data residency requirements

\begin{itemize}[noitemsep]
    \item \textbf{Regional Deployment}: EU-only CVD instances for GDPR-regulated data
    \item \textbf{Data Locality Tags}: Metadata tagging to enforce regional query routing
    \item \textbf{Cross-Region Search}: Federated queries across regional indexes (Enterprise tier)
    \item \textbf{Right to Deletion}: Efficient per-document deletion without index rebuild
\end{itemize}

\subsubsection{Supported Regions}

\begin{itemize}[noitemsep]
    \item \textbf{US}: AWS us-east-1, us-west-2; GCP us-central1

    \item \textbf{APAC}: AWS ap-southeast-1; GCP asia-east1 (roadmap)
\end{itemize}
